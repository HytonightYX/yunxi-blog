memo 优化渲染性能

React的做了个什么事情呢，用数学公式来表达，就是$UI=f(data)$

例如：

{data: 1} => <div>1</div>

如果data变为2呢？这时便重新render，

{data: 2} ==render ==> <div>2</div>

那么，会不会有这个情况，data没改变，但是也会触发组件的重新渲染呢？如果是，那么这是不是一种性能的浪费呢？答案是肯定的。

你看，第一次打印是首次渲染，第二次打印则证明了父组件的Re-Render也会导致子组件的Re-Render，但是子组件其实没有必要渲染。下面给出两种方案

1.生命周期函数 shouldComponentUpdate

意思是，只有当传入的props的name属性改变了，才触发Foo的重渲染。如果你试过，你会发现控制台确实只打印一次“Foo rendered”了

2.PureComponent

React.PureComponent 与 React.Component 很相似。两者的区别在于 React.Component并未实现 shouldComponentUpdate()，而 React.PureComponent 中以浅层对比 prop 和 state 的方式来实现了该函数。

这种方式会有两个陷阱，关键就是浅层对比四个字
